# Python スターターテンプレート

このリポジトリは、Pythonで新しいプロジェクトを始める時の「下敷き」として使えるテンプレートです。初心者でも簡単に使えるように、ログの設定、環境変数の管理、外部サービスとの連携などの基本的な機能を最初から準備しています。

## 🌟 このテンプレートの特徴

- **ログ機能**: プログラムの動作を記録したり、エラーを詳しく表示したりできます。デバッグ（問題解決）がしやすくなります。
- **設定管理**: 秘密の情報（APIキーなど）を安全に扱う仕組みがあります。環境変数を使って管理します。
- **外部API連携**: Rakuten APIやTwitter APIなどの外部サービスとつながる準備ができています。
- **汎用性が高い**: 不要な部分を削除したり、新しい機能を追加したりして、自分好みにカスタマイズできます。
- **初心者向け**: コメントが詳しく書いてあり、なぜその処理が必要かがわかります。

## 📁 ファイルの説明

### `TMP_main.py`
メインのプログラムファイルです。ここに自分の処理を書いて拡張します。
- ログの初期設定
- 環境変数の読み込み
- 定数の定義（例: APIのURLやTwitterアカウント情報）
- 基本的な処理の枠組み（エラーハンドリングなど）

初心者向けポイント: このファイルの冒頭で変数をまとめて宣言しているので、全体の構造がわかりやすいです。`try-except`を使ってエラーが起きてもプログラムが止まらないようにしています。

### `logging_config.py`
ログの設定を管理するファイルです。
- ログの出力形式（日時、メッセージなど）を決めます。
- 画面（コンソール）とファイルの両方に出力できます。
- ログファイルが大きくなりすぎないように、自動でファイルを分割します。

初心者向けポイント: ログレベル（DEBUG, INFO, WARNINGなど）を理解すると、デバッグが楽になります。たとえば、開発中はDEBUGレベルで詳しく表示し、本番ではINFO以上だけ表示するようにできます。

### `config.py`
設定をまとめたファイルです。
- 環境変数（.envファイル）からAPIキーなどの秘密情報を読み込みます。
- 定数（変更しない値）を定義します。
- Twitter APIやRakuten APIの認証情報を管理します。

初心者向けポイント: 秘密情報は直接コードに書かず、環境変数で管理することで、安全です。`.gitignore`ファイルで、これらの情報をGitにアップロードしないようにしています。

### その他のファイル
- `logs/`: ログファイルが保存されるフォルダ（自動作成されます）。
- `.gitignore`: Gitで管理したくないファイルを指定（例: 秘密ファイルやログファイル）。
- `README.md`: この説明ファイル（あなたが今読んでいるもの）。

## 🚀 使い方

### 1. 準備（環境設定）
1. このリポジトリを自分のパソコンにダウンロード（クローン）します。
2. Pythonがインストールされているか確認してください（なければインストール）。
3. 必要なライブラリをインストールします。ターミナルで以下のコマンドを実行：
   ```
   pip install python-dotenv requests-oauthlib
   ```
   （他にも必要なライブラリがあれば追加してください。）

4. `.env`ファイルを準備します。`.env.example`ファイルがある場合は、それをコピーして`.example`を削除するだけで`.env`ファイルになります。その後、必要な環境変数を設定します。例:
   ```
   APP_ID=あなたのRakuten APIのアプリID
   AFFILIATE_ID=あなたのRakutenアフィリエイトID
   TW_SUB_CK=TwitterのConsumer Key
   TW_SUB_CS=TwitterのConsumer Secret
   # 他のAPIキーも同様に設定
   ```
   ※ `.env`ファイルは`.gitignore`でGit管理から除外されるので、安全です。

### 2. 実行方法
1. `TMP_main.py` を開いて、自分の処理を追加します。
2. ターミナルで以下のコマンドを実行してプログラムを動かします：
   ```
   python TMP_main.py
   ```
3. ログが画面とファイル（`logs/my_app.log`）に出力されます。

### 3. カスタマイズのヒント
- 不要な部分（例: 特定のAPI連携）を削除して軽くする。
- 新しい機能を追加（例: データベース連携やWebサーバー）。
- コメントを参考に、なぜそのコードが必要かを理解しながら編集。
- エラーが起きたら、`traceback`が詳しく教えてくれるので、修正しやすくなります。

## ⚠️ 注意事項
- **秘密情報の扱い**: APIキーなどは必ず環境変数で管理し、直接コードに書かないでください。
- **ログファイル**: `logs/`フォルダにログが溜まるので、定期的に整理しましょう。
- **初心者向けアドバイス**: 最初は小さな変更から。わからない部分があれば、コメントを読んでみてください。プログラムは「なぜそうするのか」を考えながら書くと上達します。
- **拡張の際**: このテンプレートをベースに、自分だけのプロジェクトを作ってみましょう！
